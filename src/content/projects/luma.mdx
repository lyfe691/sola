# One Interface, Every Model

Most AI apps lock you into a single provider. Luma doesn't — bring your own API keys and switch between Claude, GPT, Gemini, Grok, Mistral, DeepSeek, and more without leaving the conversation. The interface stays minimal: a sidebar for your chats, a model picker, and a clean input area. No clutter, no upsells.

## What You Get

The home screen opens with a clean prompt input and a model selector. Pick any model from 14+ providers — the selector groups them by provider and shows capabilities (reasoning, vision, tools, fast) at a glance.

Every conversation is a full tree, not a flat list. Edit a previous message and Luma branches the conversation, keeping both paths. Navigate between branches with the branch control, or switch back to the original thread anytime.

## The Chat Tree

Conversations are stored as a tree of message nodes, not a flat array. Each node has a `parent_id`, enabling full branching — edit a message and a new branch is created. The active path is tracked via `active_leaf_id` on each chat session.

`chat-tree.ts` handles the core logic: `buildGraph()` creates an in-memory adjacency map from the flat node list, and `buildTimelineMessages()` walks from root to the active leaf to produce the linear message array for rendering.

## Providers & Models

Luma supports 14 providers out of the box. Each uses the appropriate Vercel AI SDK adapter — native SDKs for OpenAI, Anthropic, Google, xAI, Mistral, Cohere, and Alibaba, plus an OpenAI-compatible adapter for DeepSeek, Kimi, GLM, StepFun, Inception, and MiniMax.

Adding a new model is a single entry in `providers.ts`. The model ID format is `"provider:model-name"` (e.g., `"anthropic:claude-opus-4-6"`), and the chat route dynamically creates the right SDK client based on the provider type.

## Security

API keys are encrypted with AES-256-GCM before hitting the database. The encryption key lives server-side only — keys are decrypted per-request in the chat API route, never exposed to the client. All database access goes through Supabase RLS policies scoped to `auth.uid()`.

## Tools

The model has access to two built-in tools:
- **Web search** via Tavily — up to 2 calls per response, 5 results each, rendered as inline citations
- **Image generation** via Grok Imagine, DALL-E 3, or Google Imagen 3 — generated images appear inline in the conversation

Prompt modes (`search`, `image`, `study`) adjust the system prompt and tool priority based on what you're trying to do.
